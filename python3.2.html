3.2 Числа с плавающей точкой

Числа с плавающей точкой

Мы с вами уже встречались с такими числами, когда использовали /. Давайте рассмотрим их подробнее.

Их можно записывать множеством способов:

n1 = 5. # 5.0
n2 = 5.0
n3 = 05.0 # 5.0

n4 = 5e0 # 5.0
n5 = 5e1 # 50.0
n6 = 5e3 # 5000.0

n6 = 1_000_000.0
n7 = 1.0_0_1

n8 = -5.3
n9 = .56 # 0.56

Вы так же можете выполнять любые арифметические операции с этими числами.

f1 = 2.4
f2 = 3.2
print(f1 + f2 * f1) # 10.08

Если вы работаете одновременно с целыми и числами с плавающей точкой, то произойдет автоматическое приведение к значению с плавающей точкой.

print(3 + 2.4) # 5.4
print(3 + 2.0) # 5.0

Аналогично целочисленному типу для чисел с плавающей точкой требуется явное приведение, если вы ходите вводить их с клавиатуры.

f1 = float(input())
f2 = float(input())
print(f1 + f2 * f1)

Почему с float может быть опасно работать?

Давайте попробуем решить простую задачу с помощью Python. Нужно сложить числа 0.1, 0.2 и 0.3.

print(0.1 + 0.2 + 0.3) # 0.6000000000000001

Откуда взялась эта единица? А что, если продолжить вычисления, вдруг эта единица пропадет?

s = 0.1 + 0.2 + 0.3
s += 1.1
print(s) # 1.7000000000000002

Такая погрешность объясняется внутренним несовершенством архитектуры компьютера. Дело в том, что в памяти числа с плавающей точкой хранятся в приближенном состоянии в виде сумм отрицательных степеней двоек.

Отсюда возникают различные казусы, которые начинающие разработчики могут упустить, например:

a = 0.1 + 0.2 + 0.3
# == это логический оператор сравнения, мы с ним познакомимся в уроке про условия
print(a == 0.6) # False

Мы видим, что с математической точки зрения число a == 0.6, но с практической программа выдает нам False, что означает неравенство чисел.

Поэтому такие сравнения нужно обязательно проводить другим образом:

a = 0.1 + 0.2 + 0.3
# == это логический оператор сравнения, мы с ним познакомимся в уроке про условия
print(abs(a - 0.6) <= 0.00000001) # True
Чтобы сравнить два таких числа мы подбираем очень маленькое число, которое стремится к 0. Если разность меньше этого числа, значит мы с уверенностью можем сказать, что числа равны.

Аккуратным нужно быть даже в таких случаях:

print(1.1125899906842624) # 1.1125899906842625

Другой пример опасности, которая вас может подстерегать:

print(1.11111111111111111129) # 1.1111111111111112

float обрезает числа после точки до определенного значения. Такое поведение может сломать вам расчеты.

Как избежать опасностей?


Есть несколько простых правил:

  • Если есть возможность не использовать float – не используйте его. Если в вашей задаче несколько единиц измерения – переводите все в наименьшую, чтобы не работать с дробями.

  • В случаях когда нужны точные дробные расчеты лучше воспользоваться библиотеками fractions и decimal.

Модули


Python – очень удобный для разработки язык программирования и это удобство частично связано с тем, что много классного и производительного кода уже написано другими разработчиками, а мы можем использовать его для своих целей.

Такие наборы готового кода называется модулями и некоторое их количество уже написано разработчиками языка Python. Они собрали эти модули в стандартную библиотеку языка и там находится множество полезных модулей, таких как os, random, sys, decimal, и т.д. С полным списком можно ознакомиться по ссылке внизу. Вы будете изучать все больше модулей по мере изучения языка, а сейчас необходимости погружаться в них нет.

Мы с вами научимся работать с модулями на примере очень полезного модуля random. Этот модуль позволяет генерировать псевдослучайные числа. Чтобы начать использовать его в своем коде достаточно написать команду import и его название.

import random

Если вы хотите использовать модуль, то прежде всего стоит заглянуть в его документацию, чтобы узнать какие функции позволяют решить вашу задачу. Документацию можно достаточно просто нагуглить по названию модуля.

Если у нас стоит задача сгенерировать случайное целое число от 1 до 100, то нам подойдет функция randint. Для ее использования достаточно написать название модуля и имя функции.

import random

print(random.randint(1,100))
Если вы несколько раз запустите программу, то увидите, что она выдает случайное число.

Если не хочется писать такой длинный вызов функции, то можно импортировать из модуля только ее одну.

from random import randint

print(randint(1,100))

Или сократить название модуля с помощью as.

import random as rd

print(rd.randint(1,100))

Точно так же можно изменить и название функции.

from random import randint as rint

print(rint(1,100))

Если вы пишите программу или проект в команде, то не стоит переименовывать модули и функции. Это может быть совершенно непривычным для другого разработчика и ему будет сложно вникнуть в ваш код.

Если разработчики языка Python сами написали эти модули, то почему они их вынесли в отдельное место и зачем нам нужно их импортировать в свой код? Почему нельзя было просто добавить в окружение языка, чтобы можно было просто писать так:

# код не будет работать

print(randint(1,100))

Дело в том, что каждый новый импорт инициализирует новый модуль, который забирает некоторое количество оперативной памяти для своей работы. Не импортируйте модули, которые не собираетесь использовать!

А как можно импортировать в свой код модуль, которого нет в стандартной библиотеке? Для этого нужно скачать ее из индекса. Чтобы это сделать можно воспользоваться модулем pip. По умолчанию он использует Python Package Index (PyPI).

Попробуем скачать популярный модуль для сетевых запросов requests. Напишите в терминале команду python -m pip install requests.



Если после этого программа ниже не выдает ошибок, значит вы можете использовать сторонний модуль.

import requests

decimal


Мы разобрались, что работать с float нужно осторожно из-за его неточности. Но что делать, если все-таки нужно работать с числами с плавающей точкой и никак нельзя перейти к целым числам? В этом нам поможет модуль стандартной библиотеки decimal. Сравним его точность в сравнении с float.

from decimal import Decimal

a, b = "0.1", "0.2"

print(float(a) + float(b)) # 0.30000000000000004
print(Decimal(a) + Decimal(b)) # 0.3

Decimal – это новый для нас объект. Он работает не так же как float, хоть и очень похож на него. Расчеты с Decimal точные, давайте убедимся в этом взяв примеры из предыдущей статьи про float.

from decimal import Decimal

a, b, c, d = "0.1", "0.2", "0.3", "1.1"

print(Decimal(a) + Decimal(b) + Decimal(c)) # 0.6
print(Decimal(a) + Decimal(b) + Decimal(c) + Decimal(d)) # 1.7

Decimal поддерживает множество математических операций. Некоторые из них вы можете увидеть ниже, а за полным списком всегда идите в документацию к модулю!

from decimal import Decimal

a, b = Decimal("12.1"), Decimal('203.2')

print(a + b) # 215.3
print(a - b) # -191.1
print(a * b) # 2458.72
print(a / b) # 0.05954724409448818897637795276
print(a // b) # 0
print(a % b) # 12.1
# корень
print(a.sqrt()) # 3.478505426185217265198782899

Точность можно настроить с помощью getcontext().prec.

from decimal import Decimal, getcontext

getcontext().prec = 4

a, b = Decimal("12.1"), Decimal('203.2')

print(a / b) # 0.05955

На этом моменте у вас может возникнуть резонный вопрос: если Decimal такой классный, вычисляет точно, то почему разработчики языка не сделали его стандартным типом, а придумали вместо него неточный float? Ответ на вопрос:

  • Decimal занимает намного больше оперативной памяти

  • Decimal работает намного медленнее float

  • точность вычислений не всегда в приоритете:)

    • Поэтому в разработке мы опираемся всегда на наши потребности – если нам важна точность, то мы используем Decimal. Если нам важна скорость и оптимальность по памяти, но в точности можем немного сэкономить – используем float.
