5.3 Стек, очередь, куча и дек

Стек

Стек – это структура данных, которая работает по принципу последним пришел – первым вышел (LIFO). Ее можно сравнить с Ханойской башней. Чтобы добраться до основания нужно вынуть все верхние элементы.



Стек используется для таких задач, как:

  • Для создания более сложных структур данных, таких как графы и деревья

  • Для функционала отмены действий (как в IDE Ctrl+Z)

  • И многих других задач...

В Python самой логичной реализацией стека будет использование списка.

stack = []

stack.append(1)
stack.append(2)
print(stack) [1,2]

element = stack.pop()
print(element) # 2
print(stack) # [1]


Так же стек еще реализуют на односвязном списке, но для такой реализации требуется знание ООП.

Очередь


Очередь – это структура данных, которая работает по принципу последним пришел – последним вышел (FIFO). Принцип такой же как в жизни. Очередь может быть к врачу, на кассе в магазине или в других местах.



Очередь используется для таких задач:

  • На принципах очереди построены отложенные и фоновые задачи;

  • Если нужно совершить какие-то действия в порядке их поступления;

В Python очередь можно реализовать при помощи deque из модуля collections. Списки уже не помогут, так как удаление из начала списка влечет за собой перестройку списка, что делает такую реализацию очень медленной.

from collections import deque

queue = deque()

queue.appendleft(1)
queue.appendleft(2)
print(queue) # deque([2, 1])

f = queue.pop()

print(f) # 1


Очередь так же можно организовать на связном списке и двух стеках.

Дек


Двусторонняя очередь – структура данных, в которой добавление осуществляется как с начала, так и с конца списка.

Дек в основном используется для реализации более сложных структур данных.

В Python дек реализуется посредством deque.

from collections import deque

numbers = deque([1, 2, 3, 4])
numbers.popleft() # 1
numbers.popleft() # 2

print(numbers) # deque([3, 4])

numbers.appendleft(2)
numbers.appendleft(1)
numbers # deque([1, 2, 3, 4])


Куча



Куча с максимумом в вершине – это структура данных, которая представляет из себя дерево, в котором любая вершина всегда больше чем ее потомки. Преимущество кучи перед обычным списком и методом max кроется в том, что извлечение из кучи максимального элемента происходит за константное время (O(1)), а метод max работает за O(n).



В примере выше 100 больше чем 19 и 36. А 19 больше чем 17 и 3, как и 36 больше чем 25 и 1.

Существует так же обратный вариант этой куче – куча с минимумом в вершине.

Кучи используются в основном для реализации очереди с приоритетами.

Очередь с приоритетами – это такая очередь, где у каждого элемента есть свой приоритет, который влияет на то, в каком месте очереди он находится.

Пример из жизни – рабочие задачи. У каждой такой задачи есть свой приоритет, и работник опираясь на него решает, какую ему задачу брать следующей.

В Python есть модуль heapq, который реализует очередь с минимумом в вершине.

from heapq import heappush, heappop

heap = []

heappush(heap, 12)
heappush(heap, 7)
heappush(heap, 24)
heappush(heap, 11)
heappush(heap, 50)
heappush(heap, 6)

print(heap) # [6, 11, 7, 12, 50, 24]
heappop(heap) # 6
heappop(heap) # 7
heappop(heap) # 11


Очередь с приоритетами можно реализовать при помощи кортежей.

from heapq import heappush, heappop

heap = []

priority = {
"critical": 0,
"blocking": 1,
"common": 2
}

heappush(heap, (priority["critical"], 10))
heappush(heap, (priority["critical"], 12))
heappush(heap, (priority["common"], 100))
heappush(heap, (priority["blocking"], 20))
heappush(heap, (priority["blocking"], 32))
heappush(heap, (priority["common"], 10))

heappop(heap) # (0, 10)
heappop(heap) # (0, 12)
heappop(heap) # (1, 20)
heappop(heap) # (1, 32)
heappop(heap) # (2, 10)
