4.3 Работа со строками

Строковые преобразования

В предыдущих уроках мы подробно рассмотрели основные типы данных (int, float, bool) и оставили без должного внимания тип str. Конечно, мы им активно пользовались каждый раз, когда вызывали функции input и print, но этот тип данных обладает богатым функционалом, которой может быть полезен Backend разработчику.

Когда мы выводим числа при помощи функции print на экран, они автоматически переводятся в тип str, однако, можно это сделать и принудительно, если написать print(str(123)). В большинстве случаев так делать не нужно, но иногда встречаются сложные объекты, которые без принудительных преобразований не вывести.

Точно так же работает и наоборот, если например имеется строка '123' и мы знаем, что в этой строке содержится число, то мы можем привести его к нужному типу, вызвав функции int или float.

a = int('123') # 123


Почему нельзя оставить строковый тип и работать с ним без лишних операций?
Дело в том, что для строк в случае если вы их пытаетесь сложить, или конкатенировать, предусмотрено другое поведение.

a = '123'
b = '23'
print(a + b) # выведет 12323
print(int(a) + int(b)) # выведет 146

Работа со строками


Самый часто используемый метод при работе со строками – расчет длины. Посчитать ее можно при помощи функции len.

print(len('abcd')) # выведет 4


Строка – последовательность символов. В этой последовательности можно обращаться к определенным символам. Нумерация начинается с 0.

s = "backend"

print(s[0]) # b
print(s[3]) # k


Чтобы обратиться к последнему символу можно написать

print(s[len(s)-1]) # d


В программировании смотреть строки с конца – довольно частая операция, поэтому для этого случая придумали синтаксический сахар, и можно писать просто -1 без вызова функции len(s).

print(s[-1]) # d
print(s[-3]) # e


Если вы захотите поменять какой-то из символов строки, то у нас для вас плохие новости.

Дело в том, что основные типы в Python являются неизменяемыми (immutable). И код ниже выкинет ошибку.

s = "backend"
s[0] = "B" # выбросит исключение


Если все-таки нужно сделать замену какой то переменной, то можно создать копию строки и заменить в ней некоторые части. Для этого можно воспользоваться срезами, о которых мы поговорим в следующем шаге либо воспользоваться методами строк.

Для этого типа данных так же предусмотрены арифметические операции. Операция + складывает две строки и превращает их в одну, а * копирует исходную строку на заданное количество раз.

s = "back"

print(s * 2) # backback
print(s * 3) # backbackback
print(s + "end") # backend


Строки можно сравнивать друг с другом. Сравнение происходит в лексикографическом порядке (по алфавиту).

print('abc' > 'bca') # False
print('bba' < 'bbb' ) # True

Срезы


Срез строки – это извлечение из данной строки ее подстроки. Чтобы создать базовый срез нужно указать стартовый индекс и конечный, причем правая граница не включительно (прямо как в функции range).

s = "backend"

print(s[0:4]) # back
print(s[4:7]) # end


Всего есть 5 способов создания срезов. Мы можем опускать какие-то из параметров среза.

s = "backend"

print(s[0:4]) # back (от символа с индексом 0 до 4 не включительно)
print(s[1:]) # ackend (от 1 до конца строки)
print(s[:3]) # bac (от начала строки до 3 не включая)
print(s[0:7:2]) # bce (от начала до 7 не включая с шагом 2)
print(s[:]) # копия строки


И не забываем про существование отрицательных индексов строк.

s = "backend"

print(s[:-1]) # backen (от начала до конца без последнего)
print(s[-4:]) # kend (от -4 до конца строки)


ord и chr


Функции ord и chr помогают преобразовать символ в числовое представление и обратно.

print(ord('a')) # 97
print(chr(97)) # a


Числа, которые берутся от преобразования символов берутся из кодировки по умолчанию UTF-8 или Unicode. У этих кодировок есть разные разновидности и виды, но юникод является самой универсальной из них. Буквы в кодировке расположены по алфавиту, поэтому можно сдвинуть любой символ.

s = 'b'
print(chr(ord(s)+2)) # d

Методы строк


У строк есть множество полезных методов, один из них вам уже приходилось использовать в одной из задач урока. Этих методов много, и каждый из них решает свою задачу.

s = "frontend"

# возвращающий количество вхождений указанной подстроки (или символа) в данную строку
print(s.count('n')) # 2

# метод, проверяющий, что строка начинается с заданной подстроки
print(s.startswith('front')) # True

# метод, создающий копию строки с заглавной первой буквой
print(s.capitalize()) # Frontend


Рассмотрим подробнее лишь некоторые из этих методов, а вы когда будете работать со строками всегда открывайте список методов и смотрите нет ли уже готового метода, который решает вашу задачу.

replace


s = "backend or frontend"

# Заменяет все символы в строке на заданные, создавая копию
print(s.replace(' ', ' <=>')) # backend <=>or <=>frontend
print(s.replace(' ', ' <=>', 1)) # backend <=>or frontend (заменить только один раз)


find


s = "abacaba"

# возвращает индекс найденной подстроки
s.find('b') # 1
s.find("bac") # 1
s.find("d") # если подстрока не найдена, то вернет -1

s.rfind('b') # 5 поиск ведется справа строки


find работает на основе алгоритма Бойера-Мура-Хорспула, что дает ему среднюю сложность O(t) и худший случай О(pt). Если вы пишите какой-то классный сервис, где важно что поиск подстроки в строке будет работать максимально быстро, то стоит присмотреться и к другим алгоритмам поиска подстроки в строке

strip


s = "\n\n abc \n\n\t\n"

# чистит строку от пробелов и специальных символов с начала строки и с конца
print(s.strip()) # abc
print(s.lstrip()) # abc \n\n\t\n
print(s.rstrip()) # \n\n abc


split и join


Эти методы тоже очень важны, их мы рассмотрим подробнее, когда будем изучать списки.

Форматирование строк


Форматирование – приведение строки к какому–то заданному формату.

В Python существует три способа форматирования:

      1. f-строки – современный способ

      2. Метод format

      3. Ручное форматирование (%)

Что из себя представляет форматирование f-строкой?


name = "Ivan"
grade = "Professor"

s = f"His name is {name} and he has a grade {grade}!"
print(s)

# His name is Ivan and he has a grade Professor!


Вы можете заметить, что такую строку можно получить и без форматирования, уже известными методами конкатенации и другими, однако все-таки это не так удобно, нежели чем создать формат, в который уже будут подставляться переменные из программы.

f-строки позволяют делать многие вещи, помимо обычной подстановки в шаблон. К примеру вы можете форматировать цифры в различных системах счисления.

# переводит в 2-ю систему счисления
print(f"{123:b}") # 1111011
# переводит в 8-ю систему счисления
print(f"{123:o}") # 173
# переводит в 16-ю систему счисления
print(f"{123:x}") # 7b
# обозначение 10-ной системы счисления
print(f"{123:d}") # 123
# перевод двоичного числа в 10 систему счисления
print(f"{int('1010', 2):d}") # 10


Помимо целых чисел можно так же форматировать дробные.

num = .34567 # 0.34567

# округлить число до двух знаков после запятой
print(f"{num:.2f}") # 0.35
# перевести число в проценты
print(f"{num:%}") # 34.567000%
print(f"{num:.2%}") # 34.57%


Существуют так же различные консольные выравнивания, управление сепарацией и так далее. Очень подробно возможности f-строк рассмотрены тут.

format и ручное форматирование


Для метода format и % работают все те же методы форматирования, что и для f-строк.

# метод format
number = 23.8589578
print("{:.2f}".format(number)) # 23.86
# ручная подстановка
print("%0.2f - %e" % (number, number)) # 23.86 - 2.385896e+01


Вы закономерно можете спросить: "А зачем же тогда нужно целых 3 различных метода форматирования, если весь функционал есть в f-строках?".

На самом деле в мануальной подстановке авторы курса преимуществ точно не видят, она пришла из Python 2. И ее используют только программисты, которые уже давно программируют на Python.

А вот метод format содержит в себе одно важное преимущество.

Дело в том, что в модульных проектах не всегда бывает так, что переменная находится в том же окружении, что и строка-шаблон.

# у нас нет переменной name

s = f"Моё имя - {name}"

# Traceback (most recent call last):
# File " <stdin >", line 1, in <module >
# NameError: name 'name' is not defined


В случае с format можно определить шаблон в каком удобно месте и уже при выводе передать в него аргументы.

# у нас нет переменной name

s = "Моё имя - {name}"

print(s.format(name='Антон')) # ошибки уже не будет


r-строки и b-строки


r-строка – строка которая должна печататься в неизменном виде. Что это значит?

print('hello\nmister')
# hello
# mister

print(r'hello\nmister') # hello\nmister
name = 'john'
print(fr'hello\nmister {name}') # можно комбинировать


Когда это может быть полезно? Например если мы работаем с файловой системой Windows. Проще всего конечно же просто поставить r перед строкой.

print('Файл C:\windows\system32\config\today\nesquick недоступен')
# Файл C:\windows\system32\config oday
# esquick недоступен

# fixed 1 способ
print(r'Файл C:\windows\system32\config\today\nesquick недоступен')
# Файл C:\windows\system32\config\today\nesquick недоступен

# fixed 2 способ
print('Файл C:\\windows\\system32\\config\\today\\nesquick недоступен')


b-строки же в свою очередь это последовательность байтов. С бинарными строками вы будете работать при работе с файлами, изображениями и так далее.
