4.1 Условия

Условия

Переходим к условиям! Раньше наши программы имели линейную структуру – прочитай числа и сделай с ними какие-то преобразования. Но часто случается так, что в зависимости от входных данных нужно делать совершенно разные вещи.

Например, нам нужно написать программу, которая определяет, что число четное, или же программу, которая получает пароль пользователя, и если длина вводимого пароля меньше 5, то сказать пользователю, что пароль слишком короткий.

Сами по себе условия делятся на простые и сложные. А в результате же условие может быть либо Истиной, либо Ложью.

Для того, чтобы оперировать понятиями истины и лжи в Python есть тип bool. Мы уже знакомились с этим типом данных, и в этом уроке начнем им плотно пользоваться. Число с таким типом может принимать всего два значения - True и False.

Простые:

  • Число равно 5
  • Число делится на 2
  • Число не делится на 2
  • Строка равна «Валера»

Сложные:

  • Число делится на 2 и Число делится на 3
  • Возраст < 18 и Возраст> 12
  • Завтрак == бутерброд или Завтрак == чай

Как тут можно догадаться, сложные запросы состоят из нескольких простых.

В Python существует всего 6 простых логических операторов:

  • < Меньше (Первый операнд меньше второго)
  • > Больше (Первый операнд больше второго)
  • <= Меньше или равно
  • >= Больше или равно
  • == Равенство
  • != Неравенство

Например a == b, a != b, и так далее.

Операторы сложных условий:

  • and – логическое «И» (Конъюнкция) – если True and True, тогда итоговое выражение – True, в остальных случаях False;
  • or – логическое «ИЛИ» (Дизъюнкция) – если False or False, тогда False, в остальных случаях True;
  • not – отрицание (Инверсия) – not True == False – инвертирует значение bool на противоположное.

Например: a == b and c == d, a != 5 or a >= 10 и т.д.

Страшные слова Конъюнкция, Дизъюнкция и Инверсия появились из алгебры логики, которую придумал математик Джордж Буль. В его честь алгебру логики еще называют "булевой алгеброй". Там есть много разных логических операторов, из которых основными являются первые 16. Но обычно в backend разработке пользуются только первыми тремя, поэтому остальные мы рассматривать не будем.

Давайте попрактикуемся в написании логических условий.

1) Условие: число x находится в строгом промежутке от 5 до 15.

5 < x and x < 15 или 5 < x < 15. Вторая запись является более предпочтительной, потому что лучше читается.

2) Условие: число x нечетное.

x % 2 == 1 или можно сократить до x % 2. Дело в том, что если число x – нечетное (к примеру = 3), тогда 3 % 2 = 1, а 1 в языке Python приравнивается к True. Поэтому в дополнительной проверке уже не остается особого смысла.

3) Условие: год является високосным (не делится на 100 и делится на 4 или делится на 400)

year % 100 != 0 and year % 4 == 0 or year % 400 == 0

Условные операторы


Перейдем к вещам сложнее – логическим конструкциям. Часто бывает так, что нам в зависимости от какого-то результата логического условия нужно делать различные действия. Для этого есть условные операторы.

Давайте напишем программу, которая делает число положительным:

x = int(input())
if x < 0:
x = -x
print(x)


В Python обязателен отступ в 4 пробела после начала конструкции.

Что делать, если требуется выполнить что-то в случае, если условие – ложное. Для этого есть ключевое слово else.

x = int(input())
if x < 0:
print(-x)
else:
print("Число уже положительное:", x)

Давайте несколько усложним, и напишем программу, которая определяет, что число больше 5, но меньше 25 и делится на 4. Первая реализация которая приходит в голову, может быть такой:

num = int(input())

if num > 5:
if num < 25:
if num % 4 == 0:
print("Это наше число")
else:
print("Это не наше число")


Но в этой программе есть большой недостаток – она не печатает ничего для случаев, когда num = 26 или num = 4. Это происходит потому, что else относится только к одному оператору if – под которым он расположен, а не ко всем сразу. Поэтому можно исправить этот недочет, и программа будет работать как надо.

num = int(input())

if num > 5:
if num < 25:
if num % 4 == 0:
print("Это наше число")
else:
print("Это не наше число")
else:
print("Это не наше число")
else:
print("Это не наше число")


Но тут возникает другая проблема: если нас попросят поменять строку "Это не наше число" на что-то другое, то нужно будет менять ее 3 раза. Это очень не удобно, и по этой причине следует стараться избегать вложенные конструкции, если это возможно.

Правильным же решением будет использование сложных условий.

num = int(input())

if 5 < num < 25 and num % 4==0:
print("Это наше число")
else:
print("Это не наше число")


Напишем уже другую программу, которая будет определять в какой четверти находится точка в декартовой плоскости.

x = int(input())
y = int(input())
if x > 0:
if y > 0: # x > 0, y > 0
print("Первая четверть")
else: # x > 0, y < 0
print("Четвертая четверть")
else:
if y > 0: # x < 0, y> 0
print("Вторая четверть")
else: # x < 0, y < 0
print("Третья четверть")


Тут мы видим уже другую проблему использования вложенных конструкций – ухудшение читаемости кода. Для дополнительных условий в случае, если первое равно False существует оператор elif.

x = int(input())
y = int(input())
if x >= 0 and y > 0:
print("Первая четверть")
elif x >= 0 and y < 0:
print("Четвертая четверть")
elif y > 0:
print("Вторая четверть")
else:
print("Третья четверть")

Тернарный и моржовый операторы


Для улучшения читаемости кода в Python были введены два оператора – тернарный и моржовый.

Тернарный оператор нужен для случаев, когда нужно присвоить переменной какое-то значение в зависимости от определенного условия, например программу ниже можно переписать:

x = int(input())
if x % 2 == 0:
y = x // 2
else:
y = x * 2

На такой:

x = int(input())
y = x // 2 if x % 2 == 0 else x * 2


Моржовый оператор в Python нужен для присваивания значений переменным и дальнейшим использованием их для каких-то действий.

Например, код ниже можно переписать:

x = int(input())
if x % 2 == 0:
print(x ** 2 + 5)
else:
print(x ** 3 - 7)


На такой:

if (x := int(input())) % 2 == 0:
print(x ** 2 + 5)
else:
print(x ** 3 - 7)


Наиболее всего преимущества моржового оператора раскрываются в циклах.

Конструкция match/case


Нововведение Python 3.10 это конструкция match/case, который сильно в некоторых моментах улучшает читаемость кода.

command = "print"
argument = "hello"

match command:
case "print":
print(argument)
case "decorate":
print("decorated", argument)
case _: # аналог else с использованием анонимной переменной (переменная принимает любое значение)
print("error")

# hello


Не стоит путать match с очень популярным switch, которого в Python нет. match намного мощнее.

command = "say"
argument = "privetik"

match command:
case "print" | "write" | "say": # это аналог оператора or
print(argument)
case "decorate":
print("decorated", argument)
case _:
print("error")

# privetik


Или например можно мэтчить сразу по двум аргументам.

command = "decorate"
flag = "decorate your: "
argument = "privetik"

match command, flag:
case "print" | "write" | "say", "console":
print(argument)
case "decorate", _ as f:
print(f, argument)
case _:
print("error")

# decorate your: privetik


Лучше всего этот оператор раскрывается при работе с кортежами, списками и другими составными структурами данных.
